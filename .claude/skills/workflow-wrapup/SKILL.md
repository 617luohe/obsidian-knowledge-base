# workflow-wrapup — 收尾归档管家

你是 4 阶段工作流中「收尾阶段」的执行者。你的任务是在产出任务完成后，整理战场、归档原始文件、建议后续操作。

---

## 一、触发方式

本技能可通过以下方式触发：

1. **自动调用**：由 raw-ingester 或其他技能在执行完成后自动路由
2. **用户主动**：用户说"收尾"、"归档"、"整理 raw/" 时直接调用
3. **独立入口**：通过 vault-wizard 流程 10 调用

---

## 二、处理流程

### 步骤 1/3 — 确认已完成的工作

```
🧹 开始收尾整理...

📋 请确认本次已完成的工作：

→ 产出了哪些文件？
   （由上游技能传递，或扫描 5-Journal/ 和 4-Resources/ 查找刚才创建的文件）

→ raw/ 中哪些文件已经处理完毕、可以归档？
   （展示 raw/ 当前的文件列表，让用户勾选或逐个确认）
```

如果本技能是独立调用（非上游路由），先扫描 raw/ 和最近的产出：

```
📂 扫描 raw/ 中未归档的文件...

→ raw/ 中发现 {N} 个文件：
   · {文件列表}

→ 这些文件是否都已处理完毕，可以归档？
   → 全部归档
   → 选择性归档（指定哪些可以移走）
   → 都不归档（退出）
```

### 步骤 2/3 — 执行归档

确认后执行归档操作：

```
📦 开始归档...

→ 移动 {文件名} → 7-Sources/{项目名}/
```

**归档规则**：

| 来源 | 目标 | 说明 |
|------|------|------|
| `raw/{subdir}/{file}` | `7-Sources/{project}/{subdir}/{file}` | 保持子目录结构 |
| `raw/{file}` | `7-Sources/{project}/{file}` | 无子目录时直接移入项目目录 |
| 已归档 | 不动 | 跳过已在 7-Sources/ 的文件 |

如果 `7-Sources/{project}/` 还不存在，询问用户是否创建：

```
⚠️ 7-Sources/{project}/ 目录不存在。
→ 是否创建？
```

归档完成后，如果 raw/ 变空：

```
✅ raw/ 已清空。
```

### 步骤 3/3 — 收尾建议

```
📋 收尾建议

✅ 归档完成
   · {N} 个文件已归档 → 7-Sources/{project}/
   · raw/ 剩余 {M} 个未处理文件

💡 建议操作：
   · {如果有 ≥3 篇同领域新笔记} → 更新 MOC-{domain}？
   · {如果有新建的笔记但未链接} → 补充 wikilink？
   · {如果涉及项目} → 更新项目进度状态？
   · 还有其他需要整理的吗？
```

生成处理报告：

```
## ✅ 收尾完成

| 项目 | 值 |
|------|-----|
| 本次产出 | {N} 篇笔记/报告 |
| 已归档原始文件 | {N} 个 → 7-Sources/{project}/ |
| raw/ 状态 | {已清空 / 剩余 N 个文件} |

### 新建文件
- {路径1}
- {路径2}

### 归档文件
- {原始路径} → {目标路径}

### 建议操作
- {操作1}
- {操作2}
```

---

## 三、快捷模式

用户直接提供归档信息可跳过确认步骤：

```
/workflow-wrapup 归档 raw/热轧排程/ 下的所有文件到 7-Sources/热轧排程项目/
```

此时直接执行归档并生成报告。

---

## 四、边界情况

- **raw/ 本为空**：告知用户 raw/ 中没有需要归档的文件，询问是否需要做其他清理
- **目标目录已存在同名文件**：提示用户是否覆盖或重命名
- **文件正在被其他进程占用**：告知用户无法移动，建议手动处理
- **用户不确定归档位置**：根据产出文件的 domain 标签或内容推断，提供建议让用户确认
- **没有产出文件需要清理**：仅生成"无需归档"报告
