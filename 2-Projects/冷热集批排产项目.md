# 冷热集批排产项目

> 钢铁行业热轧工序计划排程系统，归属于工作项目（非个人项目）。系统覆盖从粗轧规则排程到蚁群优化、评分评估的完整排程链路。

## 项目概述

热轧是钢铁生产流程中的核心工序。物料通过轧辊顺次轧制，**每一个辊期（roll cycle）** 是一个完整的处理单元。本系统负责自动编排热轧计划，替代传统人工排程，提升排程质量和效率。

### 系统架构（两阶段排程）

```
收池物料 → 粗轧规则排程 (CoarseRollingRuleScheduler) → 蚁群优化算法 → 评分评估 (PlanScorer)
```

| 阶段 | 方法 | 产出 |
|------|------|------|
| **粗排** | 基于 B 类邻接规则的约束满足 + 启发式搜索 | 可执行的主轧段 + 烫辊段 + 过渡段 |
| **蚁群优化** | 蚁群算法（ACO）优化物料序列 | 优化后的精细排程结果 |
| **评分** | PlanScorer 评分引擎 | 结构化评分明细（CSV） |

## 领域概念

### 物料类型

| 物料 | 说明 |
|------|------|
| **主轧材** | 辊期主要轧制物料，优先排产，数量最大化 |
| **次轧材** | 填充辊期用，主轧材数量不足时补充 |
| **过渡材** | 主轧材之间规格差异过大时使用，保证轧机平稳过渡 |
| **烫辊材** | 辊期开头使用的易轧材料，用于预热轧辊、防安全事故 |
| **可利用材** | 规格可变的物料，可通过切割改变宽/厚，充当过渡材补充 |

### 辊期结构

一个完整的辊期由三段拼接而成：

```
[烫辊段] → [过渡段] → [主轧段]
```

- **烫辊段**：新轧辊更换后，用低难度物料预热轧辊
- **过渡段**：从烫辊段到主轧段的规格过渡
- **主轧段**：核心轧制阶段，B 类邻接约束最严格

### 核心约束规则

- **B 类邻接规则**：物料相邻的基本底线约束——宽度、厚度、硬度、出钢标记必须满足可邻接条件
- **宽窄印记**：从宽到窄轧制，避免窄→宽产生的表面印记缺陷
- **同宽公里数限制**：宽度差 ≤60mm 的物料连续轧制长度受限
- **辊期开头厚度限制**：新轧辊需较厚较宽物料开场

## 粗排算法

### CoarseRollingRuleScheduler

位于 `src/coarse_rolling_rule_schedule.py`，两阶段"门禁"模式：

| 模式 | 配置 | 行为 |
|------|------|------|
| **门禁开启** (默认) | `enforce_executable_coarse_plan=true` | 主轧链必须 100% 满足 B 类规则，不可贯通则输出 0 行 + failure_code |
| **门禁关闭** | `enforce_executable_coarse_plan=false` | 回退历史行为，允许断点硬接 + 最少违例 |

### 主轧段构建流程

1. **排序**：宽度 ↓  → 厚度 ↓  → 出钢标记 ↑
2. **邻接检查**：逐对检查 `_can_adjacent(prev, curr)`，等价有向邻接矩阵
3. **无法邻接时分支处理**：
   - **宽度突破** → 次轧材 + 过渡材池 BFS 构建桥接段
   - **厚度突破** → 逐级尝试：次轧/过渡桥 → 宽窗主轧桥 → 扩展池桥 → 抽离主轧桥
   - **非宽厚原因** → 次轧+过渡桥
   - **以上均失败** → 可利用材虚拟占位
   - **全部失败** → 门禁模式：abort（不可下发）；回退模式：标记断点

### 烫辊段与过渡段

- **烫辊段**：优先非可利用材，倾向宽厚，段内不做 B 类校验
- **过渡段**：strict BFS 桥接 + 可利用虚拟；门禁模式下无 min_violation 回退
- **全链 B 复核**：拼接后跨段相邻对均须满足 B 类规则，否则清空计划

## 评分评价体系

[[热轧排程评分指标]]

23 项评分指标，覆盖六个维度：

| 维度 | 权重 | 关键指标 |
|------|------|---------|
| 物料类型 | - | 主轧材 +1/卷，可利用材 -1/卷，主轧段过渡材 -0.5/卷 |
| 轧宽趋势 | - | 宽度违规 -5/处，反跳 -1/处，尖点 -2/处，同宽集中 -1/段 |
| 轧厚趋势 | + | 厚度违规 -5/处，同厚集中 +0.5/卷 |
| 出钢标记 | + | 违规 -5/处，同标记集中 +0.5/卷 |
| 冷热坯 | ++ | 热装 +2/卷，冷热交替 -1/处 |
| 公里数 | - | 辊期超限 -1/超过，烫辊过渡超限 -0.5/超过 |

## 近期工作：排程评分集成模块

[[PRD-排程评分集成模块]]

将 `PlanScorer` 从离线评测嵌入主流程，实现排程结果的自动评分。

### 改动要点

1. **调换执行顺序**：`entrance()` 中先写库再评分（原顺序相反）
2. **评分候选数据**：`build_and_write_all_schedule_results()` 中缓存粗排和蚁群结果的 `completed_plan` / `completed_info` 副本
3. **CSV 输出**：备份目录下生成 `coarse_evaluation_score.csv` + `main_evaluation_score.csv`
4. **异常隔离**：评分失败仅 warning，不阻塞主流程
