# Grill-Me 技能介绍与使用

> 一句话定义：`/grill-me` 是一个交互式需求拷问技能，通过不断提问帮助澄清需求、发现潜在风险并优化设计方案。

## 概述

`/grill-me` 是 Claude Code 技能生态系统中的一个关键技能。它不直接给出方案或执行任务，而是扮演"魔鬼代言人"的角色，沿着决策树反复追问，直到用户与 AI 达成对计划或设计的共同理解。

## 核心功能

### 迭代问答
持续追问细节，确保需求清晰准确。不会因为用户给出了初步回答就停止，而是像剥洋葱一样层层深入。

### 分支场景探索
沿每个决策分支分析依赖关系和潜在问题。对每个选择追问"如果选 A 会怎样？选 B 呢？"帮助用户看清完整的决策空间。

### 风险识别与缓解
揭示三类隐性风险：
- **隐藏假设** — 用户和 AI 都默认的前提，但可能不成立
- **边界情况** — 正常路径之外的边缘场景
- **潜在故障点** — 可能在实施中爆炸的环节

对每个识别出的风险生成缓解措施。

### 决策追踪与记录
帮助记录每个决策的理由和可执行方案，形成可追溯的决策链。这些记录可以作为后续开发的依据。

### 代码库探索（可选）
在适用时，可以直接分析代码来回答问题，而非仅凭用户描述。使提问更有针对性、更贴近实际。

## 使用步骤

### 1. 确保 Skills 功能已启用
确认 AI 工具支持 Skills 机制，并已完成初始化设置（如 `/setup-matt-pocock-skills`）。

### 2. 调用 /grill-me
```
/grill-me 你的需求或设计方案
```
在 AI 工具中输入 `/grill-me` 并提供需求描述。描述越具体，提问越精准。

### 3. 回答 AI 追问
AI 会不断追问以下维度的问题：
- 边界条件：输入范围是什么？什么不在范围内？
- 异常路径：出错时怎么处理？降级策略是什么？
- 优先级：如果时间不够，先做什么？
- 依赖：这个决策依赖哪些前提条件？

直到需求完全澄清，AI 才会停止追问。

### 4. 生成计划
`/grill-me` 本身只提问，不会自动整理计划。完成问答后，让其他 Agent 根据问答结果执行计划或进行交叉 review。

### 5. 复核与优化
建议在执行前或执行后再次 review，检查以下内容：
- 是否有遗漏的文件或依赖？
- 测试覆盖是否足够？
- 边界情况是否都有处理方案？

## 使用场景

| 场景 | 说明 |
|------|------|
| **技术面试准备** | 模拟面试官的追问，测试方案完整性和抗压能力 |
| **架构与项目计划评审** | 提前发现设计缺陷和潜在风险，避免后期返工 |
| **复杂项目重构** | 跨模块依赖、循环依赖、大规模重构中确保需求清晰 |
| **利益相关者对齐** | 帮助团队达成共识，减少实施过程中的意外 |

## 使用建议

### 专注于追问
`/grill-me` 的核心价值在于"提问"而非"解答"。它不会急于给出方案，而是确保需求先被彻底理解。不要期望它直接帮你写代码——那是后续步骤的事。

### 与其他技能配合
- **`/grill-with-docs`**：在提问的同时读取文档和代码库，生成更精准的问题
- **`/note-composer`**：问答完成后，将决策记录整理为结构化笔记
- **`/vault-cartographer`**：将决策链链接到相关知识库笔记

### 小任务可简化
对于简单任务（如单文件修改、typo 修复），可以使用 **caveman 模式**减少不必要的分析，避免过度设计。

### 慢就是快
> 虽然提问过程看似耗时，但能显著减少后期 bug 和返工。

花 30 分钟彻底澄清需求，可能节省 30 小时的返工。这是 AI 时代需求工程的核心理念。
