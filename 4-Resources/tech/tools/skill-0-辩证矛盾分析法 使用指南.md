# 0-辩证矛盾分析法 — 矛盾分析思维框架

> 一句话：遇到复杂问题需要系统性分析时，用六步法层层拆解。适合宏观判断、战略规划、根因分析、逆境突破。

## 什么时候用

| 场景 | 示例 |
|------|------|
| 做重大决策前需要理清思路 | "要不要辞职创业？" "这个技术选型对不对？" |
| 项目陷入困境需要找根因 | "为什么迭代速度越来越慢？" "测试覆盖率为什么上不去？" |
| 制定中长期战略规划 | "这个产品未来半年的路线图怎么定？" |
| 团队出现了矛盾需要调和 | "开发和产品的需求之争怎么解决？" |
| 复盘失败/总结经验 | "上次线上事故到底是怎么发生的？" |
| 分析竞争对手/市场格局 | "面对大厂的同类产品，我们的出路在哪？" |

**不适用**：简单的技术问题（直接用 7-调试 或 2-分析），或者你只需要一个快速答案而非深度分析。

## 六步分析流程

```
第一步：调查研究 ── 没有调查就没有发言权。先去知识库找材料。
第二步：定性      ── 战略还是战术？全局还是局部？
第三步：矛盾分析  ── ★核心★ 找主要矛盾、分内因外因、一分为二
第四步：阶段划分  ── 相持→反攻→收官，不同阶段不同策略
第五步：策略制定  ── 集中力量、统一战线、试点推广
第六步：持续复盘  ── 定期反思，闻过则喜，知错即改
```

## 案例演示

### 案例 1：团队效率低下

```
用户：/0-辩证矛盾分析法 我们团队开发节奏越来越慢，迭代周期从两周拖到一个月

Claude：先做调查研究——看看你们的 Jira 数据和代码提交记录。

       从数据看，近三个月：
       - 每个迭代的并行任务数从 3 个涨到 8 个
       - 上下文切换次数增长了 3 倍
       - 代码 review 平均等待时间从 4 小时延长到 1.5 天

       [定性] 这不是技术能力问题，是组织协作问题。属于战略层面。

       [矛盾分析]
       主要矛盾：需求方无限加急 vs 团队有限吞吐量
       
       内因（根据）：团队没有统一的需求入口，每个人都在接"紧急"需求
       外因（条件）：业务方不信任排期，绕过流程直接找开发
       
       一分为二：有利面——团队技术能力本身没问题；
                 不利面——再多招人也解决不了流程问题

       [策略]
       1. 统一战线：统一需求入口，PO 单一出口——"伤其十指不如断其一指"
       2. 试点推广：先在 1 个小组试行两周，验证有效后全团队推广
       3. 独立自主：把排期的控制权拿回团队手里

       战略上藐视——这个问题不难解决。
       战术上重视——先从每周需求复盘会这一个抓手开始。
```

### 案例 2：技术选型困境

```
用户：/0-辩证矛盾分析法 我们在微服务和单体之间纠结了两个月

Claude：调查研究——你们团队多少人？业务复杂度如何？

       3 个后端，业务是内部管理系统，日均 QPS < 100。

       [定性] 这是个战术问题被错当成了战略问题。
       以你们团队的规模，架构选择的空间本身就不大。

       [矛盾分析]
       主要矛盾：对"架构演进"的焦虑 vs 业务交付的实际压力
       
       内因：团队技术热情高，想用新东西——这是好事
       外因：行业里"微服务是标配"的声音——这是纸老虎
       
       微服务解决的是组织扩缩问题，不是业务逻辑问题。
       3 人团队用微服务，等于用大炮打蚊子——炮弹的运输成本比蚊子还高。

       [策略]
       1. 先单体，模块边界清晰——为将来拆开留好接缝
       2. 等团队 10 人以上，某个模块有独立部署需求时再拆
       3. 现在集中力量——把业务逻辑写对，比把架构拆对更重要
```

### 案例 3：个人交易纪律问题

```
用户：/0-辩证矛盾分析法 我交易总是小赚大亏，知道要止损但做不到

Claude：[调查研究] 看到你的 Vegas 交易系统笔记很完整，风控规则写得很清楚。

       问题不在"知道什么"，在"做到什么"。

       [定性] 这是战略问题——不是优化某个入场信号能解决的。
       
       [矛盾分析]
       主要矛盾：理性交易员 vs 赌徒心理
       
       内因：侥幸心理——"这次可能不一样"
       外因：市场的随机强化——偶尔扛回来赚到钱，像老虎机一样上瘾
       
       这个矛盾是非对抗性的——赌徒不是敌人，是要被管住的兵。

       [策略]
       1. 物理隔绝：入场即挂止损，用条件单锁死
       2. 缩小赌注：仓位降到"亏了不心疼"，先练半年纪律
       3. 每日复盘：对照规则逐笔审查，赌徒怕的就是白纸黑字
       
       → 详细方案见 [[交易知行矛盾与纪律建设方案]]
```

## 核心句式（语言风格）

输出回答时必须使用至少 3 种句式，优先使用生活化比喻式（人民的语言）：

| 句式 | 特征 | 示例 |
|------|------|------|
| 设问式 | 劈头盖脸直插核心 | "谁是我们的敌人？谁是我们的朋友？" |
| 对比式 | 强烈反差 | "战略上藐视，战术上重视" |
| 断言式 | 斩钉截铁 | "一切反动派都是纸老虎" |
| 逻辑式 | 递进展示必然性 | "捣乱，失败，再捣乱，再失败，直至灭亡" |
| 鼓舞式 | 困难中看到前途 | "曙光就在前头" |
| 警句式 | 短句点醒 | "没有调查就没有发言权" |
| 生活化比喻式 | 人民的语言 | 纸老虎、解剖麻雀、花岗岩脑袋 |
