# use-skills — 智能技能调度器

> 一句话：用自然语言描述需求，自动匹配并执行对应的技能。不需要记任何技能名字。

## 什么时候用

| 场景 | 说明 |
|------|------|
| 第一次用这套技能体系 | 不知道有哪些技能、各叫什么名字 |
| 需求跨越多个技能 | 比如"分析代码然后重构"，需要串联 2-分析 → 6-优化 |
| 不想记命令 | 只想说人话，不想背 `/1-规划` `/4-开发` 这些路径 |

**不适用**：你已明确知道要调用哪个技能时，直接 `/技能名` 更快。

## 工作原理

```
你说 → "帮我分析并重构这个模块"
  │
  ▼
提取意图 → 分析: ✓  重构: ✓
  │
  ▼
匹配路由表 → 2-分析 + 6-优化
  │
  ▼
按依赖排序 → 先分析，后优化
  │
  ▼
依次执行 → 2-分析 输出模块地图 → 6-优化 输出重构方案
```

## 路由表速查

| 你说的关键词 | 匹配的技能 |
|-------------|-----------|
| 新项目、初始化、搭建、脚手架、创建项目 | 0-启动 |
| 规划、方案、设计、PRD、拆任务、需求分析 | 1-规划 |
| 分析、理解、概览、地图、看不懂、结构 | 2-分析 |
| 矛盾分析、辩证、战略判断、宏观、决策复盘 | 0-辩证矛盾分析法 |
| 原型、验证、试错、不确定、快速验证 | 3-原型 |
| 开发、实现、编码、写代码、TDD、功能开发 | 4-开发 |
| 审查、检查、验收、review、质量、issue | 5-检查 |
| 优化、重构、改进、架构、代码混乱、耦合 | 6-优化 |
| 调试、bug、排查、诊断、报错、崩溃、性能 | 7-调试 |
| 版本、git、提交、推送、分支、回滚、保存 | 8-版本管理 |
| 整理、交接、收尾、结束、下班、同步 | 9-最后整理 |

## 案例演示

### 案例 1：新手入门——不知道用哪个

```
用户：/use-skills 我想新建一个 Python 项目

Claude：→ 分析需求：创建新的 Python 项目
       → 匹配技能：0-启动（新项目初始化）
       → 开始执行...

       Python 版本？≥3.12？
用户：3.11
Claude：mypy 要 strict 吗？(y/N)
用户：N
Claude：Ruff 行长度？88 还是 120？
用户：120
...
Claude：项目已就绪。运行 `uv run pytest` 验证。
```

### 案例 2：多技能串联——自动编排

```
用户：/use-skills 分析一下 payment_service.py，然后帮我做重构计划

Claude：→ 分析需求：理解代码结构 + 制定重构方案
       → 匹配技能：2-分析（代码理解）→ 6-优化（重构计划）
       → 开始执行...

       [2-分析] payment_service.py 模块地图：
       ┌─────────────────────────────────┐
       │ PaymentService (line 23)        │
       │  ├── process_payment()          │
       │  ├── refund_order()             │
       │  ├── reconcile_transactions()   │
       │  └── send_notification()        │
       │  数据流：API → Service → DB      │
       └─────────────────────────────────┘

       [6-优化] 发现四个功能耦合在同一个类中。
       建议分 5 步提取，每步独立可提交：
       1. 提取 PaymentStatus 枚举
       2. 提取 PaymentProcessor（纯支付逻辑）
       3. 提取 RefundProcessor（退款逻辑）
       4. 提取 ReconciliationService（对账）
       5. 提取 NotificationService（通知）
```

### 案例 3：遇到问题——就近匹配

```
用户：/use-skills 订单接口偶尔返回 500，排查一下

Claude：→ 分析需求：排查间歇性服务端错误
       → 匹配技能：7-调试（结构化调试）
       → 开始执行...

       [阶段 1] 构建反馈回路：
       while true; do curl -s -o /dev/null -w "%{http_code}" \
         -X POST localhost:8000/payments; sleep 0.1; done
       
       运行 100 次，约 5% 返回 500（5 次失败）。

       [阶段 3] 3 条假设：
       1. 数据库连接池耗尽 → 增大 pool_size 测试
       2. 第三方网关超时 → mock 网关测试
       3. Redis 缓存穿透 → 检查命中率

       从假设 1 开始验证...
```

## 注意事项

- 多技能**按顺序串联执行**，不会同时执行两个
- 路由匹配基于关键词，表述越接近路由表中的触发词，匹配越准确
- 如果你的需求完全不匹配任何技能，会直接告知
