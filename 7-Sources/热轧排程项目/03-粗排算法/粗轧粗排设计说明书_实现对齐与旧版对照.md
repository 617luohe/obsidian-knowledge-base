### 1.1 已确认现场业务规则（默认 `enforce_executable_coarse_plan=true`）

以下与现场/安全/审计约定一致，**实现于** `CoarseRollingRuleScheduler` + `jdata["coarse_rule_schedule"].enforce_executable_coarse_plan`（默认 **true**；置 **false** 可回退历史「断点硬接、过渡最少违例」等行为）。

| 规则 | 行为 |
|------|------|
| **不可贯通则不可下发** | 主轧链在次轧/过渡/借主轧/可利用用尽后仍无法 **100% B**，或全链 B 复核失败 → **`coarse_rule_schedule_result.csv` 为 0 行**（仅列），`run()` 返回 `status=not_executable` 及 `failure_code`。 |
| **主轧段 100% B** | 主轧段内相邻（含插入桥接行）均须 `_can_adjacent.ok`；**禁止**再输出「相邻违 B + 违规分」的硬接序列。 |
| **过渡优先、可利用降级** | 厚度突破时 **先** `secondary∪transition` strict 桥，再宽窗主轧→扩池→抽离；非宽厚原因失败 **先** 次轧+过渡桥，**再** 可利用虚拟占位。 |
| **过渡段不设违 B 回退** | 在门禁打开时，过渡段 **仅** strict 桥 + 可利用虚拟；**不**再调用 `min_violation` 桥。 |
| **舍去物料** | 不可纳入合法链的料不写入结果；本系统不对仓储二次建模。 |
| **追溯与复现** | `coarse_rule_process/12_run_audit.json`（门禁开启时）记录 `failure_code`、段行数、主轧/过渡元数据；与 `00_meta.json` 中配置快照一并用于复现。 |
| **算法版本列** | 门禁开启且成功时，`粗轧规则算法版本` 为 **`v3_exec_guard`**；关闭门禁时为 **`v2_rule_based`**。 |

## 3. B 类邻接（「有向邻接」在代码中的落实）

旧版方案写「按 b 类规格构建有向邻接矩阵」；实现上**未**在主流程中物化整张 N×N 矩阵，而是对需要的点对调用 `_can_adjacent(prev, curr)`（含缓存），语义等价于**顺序对 (i→j)** 是否允许衔接。

### 3.1 判定顺序与结果结构

函数：`_can_adjacent_uncached` → 封装为 `AdjacencyCheck(ok, reasons, violation_score, detail)`。

| 检查项 | rule_b 键 | 失败原因标记（reasons 示例） |
|--------|-----------|------------------------------|
| 宽厚跳变 | `宽度`、`厚度` 表 + `_match_jump` | `width` / `thickness`；规则缺行且严格模式：`width_missing_rule` / `thickness_missing_rule` |
| 硬度跳变 | `硬度`（有列且匹配到规则时） | `hardness` |
| 过渡材规则 | `过渡材` | `trans` |
| 出钢标记规则 | `出钢标记` | `stno` |
| 缺宽厚列/非数值 | — | `missing_spec_columns` / `invalid_spec` |

**口径补充（2026-05-12）**：

- 规格差值按 **当前物料 - 下一块物料** 计算：  
  `dw = w_i - w_j`，`dt = t_i - t_j`。  
- 宽/厚/硬度跳变均按「当前规格命中的 B 类规则」进行区间判定。  
- 区间比较采用容差（epsilon）处理边界浮点误差，避免 `0.40000000000000013` 类误判。  
- `trans` 规则按「**下一块模型**所需过渡钢系包含当前模型」解释；同模型相邻默认允许。

- **`ok`**：`reasons` 为空。  
- **`strict_when_rule_missing`**：`True`（默认）时规则缺行记 missing 并计入违规分；`False` 时对宽厚用**大容差**放宽（与严格 B 类现场口径可能不一致，排查时注意配置）。

### 3.2 与旧版方案 §3 第一句对照

| 旧版 | 实现 |
|------|------|
| 按 B 类构建有向邻接并逐对检查 | 逐对 `_can_adjacent`，等价有向；大桥接时再对候选集做多点 BFS |

## 4. 主轧材排序

### 4.1 实现（`_main_sort`）

有列则依次参与排序（`mergesort` 稳定排序）：

1. **宽度**：降序  
2. **厚度**：降序  
3. **出钢标记**：升序  

无缺列则仅 `reset_index`，不额外排序键。

### 4.2 与旧版方案对照

旧版写「默认排序优先级：宽度、厚度、出钢标记」——与当前实现**一致**（宽厚均为降序，出钢升序）。

### 4.3 主轧输入池（`_resolve_main_dataframe`）

| 条件 | 主轧段输入 |
|------|------------|
| 主轧池非空 | `pools["main"]`，来源标记「主轧」 |
| 主空且 `main_pool_fallback` 允许 | 可回退为次轧池或全量 `work_df`（见配置 `main_pool_fallback`） |

回退会影响 `_main_is_fallback`，进而影响**断点后孤立段统计**用「主轧块数」还是「段长度」（见 §6.4）。

## 5. 总体执行顺序（`run()`）

**与旧版方案 §3 末尾「先主轧邻接结果，再烫辊/过渡」一致**，代码顺序为：

1. 校验 `steel_data`、宽厚列  
2. 数值化、`_split_pools`  
3. 解析主轧基表、` _main_sort`  
4. **`_compose_main_segment`** → 得到 `main_seg`；**门禁开时**若不可贯通则此处中止，后续不建烫辊/过渡  
5. **`_build_hot_segment`**  
6. **`_build_transition_segment`**（烫辊末行 ↔ 主轧首行；门禁开时无 `min_violation`）  
7. **`concat(hot, trans, main)`**，**全链 B 复核**（烫辊内部除外）  
8. 持久化 `coarse_rule_schedule_result.csv`（失败可为 **0 行**）及过程文件（含 `12_run_audit.json`）

> **排查提示**：门禁开时 **烫辊段内部** 仍不做 B 类校验；**跨段**（烫辊末起的相邻对直至主轧）由 `run()` 内全链 B 复核覆盖，见 §7.3。

## 6. 主轧段：逐步逻辑与旧版对照

实现：`_compose_main_segment(main_sorted, pools)`。  
变量：`output_rows`，`used_index`（已占用行索引），`break_positions`（断点在「当前块接入前」的索引），`virtual_rows`（可利用虚拟明细）。

初始化：`output_rows = [main_sorted 第一行]`。

对 `i = 1 … len(main_sorted)-1`：`prev = output_rows[-1]`，`curr = main_sorted.iloc[i]`，`adj = _can_adjacent(prev, curr)`。

### 6.1 直接成功

- **`adj.ok`**：追加 `curr`，来源 `main`，违规分 0，`action=direct_ok`。

### 6.2 宽度突破分支（旧版 §3·宽度）

**进入条件**：`"width" in adj.reasons` 或 `"width_missing_rule" in adj.reasons`。

| 步骤 | 实现 |
|------|------|
| 候选池 | `secondary ∪ transition`，去掉 `used_index` |
| 桥接 | `_build_bridge(prev, curr, cand, max_nodes=max_bridge_nodes, strict only)` |
| 成功 | 依次追加桥接行（`sec_or_trans`，桥接标记 True），再追加 `curr` |

与旧版「以次轧、过渡为范围做最短过渡路径」**一致**；算法为 **BFS 最短路径（strict）** + 多解时 `_pick_best_path` 厚度启发，旧版用语「贪心」在此处为**概念对应**，非同名算法。

### 6.3 厚度突破分支（旧版 §3·厚度）

**进入条件**：`"thickness" in adj.reasons` 或 `"thickness_missing_rule" in adj.reasons`。

按序尝试（成功即停止；**与「过渡优先」一致**）：

| 子步 | 候选构造 | `max_nodes` | 来源标记 |
|------|----------|-------------|----------|
| **次轧+过渡（首步）** | `secondary ∪ transition`，去掉 `used_index` | `max_bridge_nodes` | `sec_or_trans`（`action=bridge_thickness_sec_trans_first`） |
| 宽窗主轧 | `main_sorted` 中宽度 ∈ `[w_prev ± thickness_break_width_window]`，排除已用与 `curr` | `max_bridge_nodes` | `main_bridge` |
| 扩展池 | 上一步 `main_window` ∪ `sec_trans_base`，同上排除 | `thickness_break_extended_max_nodes` | `ext_main_sec_trans` |
| 抽离主轧 | `_extract_from_main`：在 **`curr` 宽度 ± 窗口** 内筛可抽离主轧，再桥接 | 同上扩展节点上限 | `extracted_main` |

与旧版「先窄窗主轧 → 扩大次轧/过渡 → 再借主轧（抽离）」**一致**；窗宽对应配置 `thickness_break_width_window`（旧版写 100，可配）。

### 6.4 非宽厚原因：次轧+过渡桥

**进入条件**：`adj.reasons` 中 **无** `width` / `width_missing_rule` / `thickness` / `thickness_missing_rule`（如仅有硬度、出钢、过渡材等）。

**数据流**：与 §6.2 相同候选池 `secondary ∪ transition`，strict `max_bridge_nodes`，成功则 `action=bridge_misc_sec_trans`。

### 6.5 可利用材虚拟（`_try_klyc_virtual_fill`）

在宽/厚分支及 §6.4 **均未成功**后执行（过渡/借主轧之后的**次要**手段）。

- 按 `klyc_max_steps` 生成虚拟宽厚序列，从 `klyc` 池选题修改宽厚，每步 `_can_adjacent` 校验，最后与 `curr` 校验。  
- 与旧版「虚拟最短过渡 + 可利用改规格插入」**一致**。

### 6.6 不可修复（`enforce_executable_coarse_plan`）

当前步 §6.2–§6.5 **均失败**时：

- **`enforce_executable_coarse_plan=true`（默认）**：`action=abort_not_executable`，`meta.status=不可下发_主轧B类不可贯通`，**立即返回空主轧表**；后续 `run()` **不拼接**烫辊/过渡，`coarse_rule_schedule_result.csv` **0 行**。  
- **`enforce_executable_coarse_plan=false`（回退）**：仍可走历史 **断点**：`break_positions` + `append_row(curr, …, adj.violation_score)`，`action=break_point`。

### 6.7 断点后孤立段处理（仅回退模式）

仅当 **`enforce_executable_coarse_plan=false`** 且出现 `break_point` 时，逻辑同历史：按 `isolate_keep_ratio` 截断或 `孤立段无法修复`。**门禁开启时不会产生断点**，故此节一般不触发。

### 6.8 与旧版方案对照（更新）

| 旧版方案描述 | 当前实现（门禁默认开） |
|--------------|-------------------------|
| 非宽厚邻接失败也应先寻过渡 | **已对齐**：§6.4 次轧+过渡桥后再可利用 |
| 间断点处不可执行 | **已对齐**：默认 **abort**，不输出违邻接主轧链 |

## 7. 烫辊段与过渡段

### 7.1 烫辊段 `_build_hot_segment`

- 取 `need = hot_segment_length` 行。  
- 池内按可利用标记分为：烫辊且**非**可利用优先；不足则补 **烫辊∩可利用**。  
- 有宽厚列时：**宽度降序、厚度降序**；**不做**段内 B 类邻接校验。  

与旧版及 `排程背景介绍` 中「优先非可利用、倾向宽厚」**一致**。

### 7.2 过渡段 `_build_transition_segment`（烫辊末 → 主轧首）

| 阶段 | 条件 | 说明 |
|------|------|------|
| 跳过 | 缺 `hot_last` / `main_first` / `trans_pool` 空 | 空过渡，`status=过渡段_skip`（以代码为准） |
| Strict 桥 | `enforce_width_direction=True`，`max_nodes = max(max_bridge_nodes, min_transition_length+1)` | 路径长度 ≥ `min_transition_length` → `过渡段_严格` |
| 可利用虚拟 | strict 失败 | 虚拟行数 ≥ 最小长度 → `过渡段_可利用材虚拟占位` |
| Min violation | 再失败 | 仅 **`enforce_executable_coarse_plan=false`**：`min_violation` 桥 → `过渡段_最少违例`。**门禁开时跳过本步**，直接 `过渡段_无可行`。 |

宽向单调约束：`_width_dir_ok`（朝主轧首块宽度方向单调）。

### 7.3 全链 B 复核（`enforce_executable_coarse_plan`）

`run()` 在拼接 `final` 后：除 **烫辊段内部**相邻对外，凡 **烫辊末→…→主轧** 的相邻对均须 `_can_adjacent.ok`。失败则 **清空计划**（0 行），`failure_code=final_chain_b_failed:*`。

### 7.4 与旧版「拼接顺序」

实现：`final = concat(烫辊, 过渡, 主轧)`，与旧版 §3 最后「拼接烫辊段、过渡段、主轧段」**一致**。

## 10. 日志排查入口与阶段含义

粗排相关运行日志统一看 `log/algorithm/schedule.log`。同一趟粗排日志会带 `run_id=<备份目录名>`；父进程超时封装与子进程算法阶段使用同一个 `run_id`，可直接按该值检索完整时间线。

建议排查顺序：

1. 先 grep `run_id=<...>`，看 `stage.start` / `stage.finish`，确认卡在父进程、数据准备、主轧、烫辊/过渡、落盘还是性能汇总。
2. 看 `stage.summary`，获取每个关键阶段的行数、状态、失败码、输出路径和耗时摘要。
3. 若 `status=not_executable`，优先打开 `coarse_rule_process/13_operation_summary.json` 与 `12_run_audit.json`；再按需阅读 `03_main_decisions.jsonl` 或 `06_transition_attempts.jsonl`。
4. `heartbeat` 只表示长耗时桥接/主轧构建仍在推进，不代表阶段完成；最终结论以 `stage.summary stage=run.finish` 或返回状态为准。

## 12. 桥接算法摘要（`_build_bridge`）

1. **`_bfs_strict_bridge`**：仅 `_can_adjacent.ok` 的边；可选 C++ `bfs_strict_bridge_dense`。  
2. 若 strict 无路径且 **`allow_fallback_min_violation=False`**（主轧宽/厚桥接默认）：返回空。  
3. 若为 `True` 且 **`enforce_executable_coarse_plan=false`**（**过渡段**第二次调用）：**`_bfs_min_violation_bridge`**。  

主轧宽/厚路径**始终不允许** min_violation；过渡段在门禁开时**不**走第二次 `min_violation`。

## 13. 逻辑偏差速查表（专为防偏差）

排查「现场理解与代码不一致」时优先核对：

| # | 主题 | 结论（默认门禁 **开**） |
|---|------|------|
| 1 | 执行顺序 | **先** 主轧合成；**仅主轧成功**后再 `hot`/`trans`/`concat` |
| 2 | 主轧排序 | 宽↓、厚↓、出钢↑ |
| 3 | 非宽厚违 B | **先** §6.4 次轧+过渡，再利用虚拟；失败则 **abort** |
| 4 | 断点硬接 | **不再出现**；回退模式见 `enforce_executable_coarse_plan=false` |
| 5 | 不可下发 | `failure_code`：`main_b_unreachable` / `no_main_material` / `final_chain_b_failed:*` |
| 6 | 烫辊段内 | **仍无** B 类校核；**跨段**由全链复核覆盖 |
| 7 | 过渡段 | **无** `min_violation`；strict + 可利用虚拟失败即无可行过渡 |
| 8 | rule_b 缺行 | `strict_when_rule_missing=false` 时宽厚仍可能「宽放」，与严 B 现场口径可能不一致 |
| 9 | 版本追溯 | CSV `粗轧规则算法版本` + `12_run_audit.json` |
