# 命名规范

## 文件命名

### 通用规则

- **使用有意义的中文名**（英文术语可保留英文）
- **空格分隔**，不使用下划线或连字符
- **避免特殊字符**（`/ \ : * ? " < > |`）
- **简洁优先**：文件名 ≤ 50 个字符

### 按目录的命名规则

| 目录 | 命名格式 | 示例 |
|------|----------|------|
| 0-Inbox | `YYYYMMDD-HHMM-主题.md` | `20260515-0930-知识库设计想法.md` |
| 1-Atlas | `MOC-领域名.md` | `MOC-机器学习.md` |
| 2-Projects | `{项目名}.md`（作为文件夹内的概述文件） | `知识库搭建.md` |
| 3-Areas | `{领域名}.md` | `健康.md` |
| 4-Resources | `{概念名}.md` | `Transformer架构.md` |
| 5-Journal/daily | `YYYY-MM-DD.md` | `2026-05-15.md` |
| 5-Journal/weekly | `YYYY-W{周数}.md` | `2026-W20.md` |
| 5-Journal/monthly | `YYYY-MM.md` | `2026-05.md` |
| 6-People | `{姓名}.md` | `张三.md` |
| 7-Sources | `{书名/文章标题}.md` | `思考快与慢.md` |
| 8-Templates | `tpl-{类型}.md` | `tpl-resource.md` |
| 9-System | 描述性名称 | `知识库体系设计.md` |

### 特殊情况

- **同名笔记**：在名称后加括号标注上下文。如 `状态管理 (React).md` 和 `状态管理 (Vue).md`
- **英文术语**：统一使用官方英文名称的原始写法。如 `Transformer.md` 而非 `转换器.md`
- **中英混排**：中文在前，英文在后。如 `检索增强生成 RAG.md`

## 属性命名

### Frontmatter 属性名

- **全小写英文**：`tags`, `created`, `aliases`
- **多词用连字符**：`last-review`, `source-type`, `review-frequency`
- **不允许空格或下划线**

### 属性值

- **日期**：统一 `YYYY-MM-DD` 格式
- **列表项**：使用 YAML 数组语法 `[项1, 项2]`
- **标签值**：`type/xxx`, `status/xxx` 等层级格式

## 链接规范

### Wikilink 格式

```markdown
[[笔记名]]                  ← 基础链接
[[笔记名|显示文本]]          ← 别名链接
[[笔记名#章节]]             ← 指向特定章节
[[笔记名#章节|显示文本]]    ← 章节别名链接
```

### 使用场景

- **正文中首次引用**：使用 `[[笔记名]]`
- **需要融入句子**：使用 `[[笔记名|显示文本]]`，如 "参见 [[Transformer架构|Transformer 模型的设计原理]]"
- **MOC 列表**：使用 `[[笔记名]] — 简短描述` 格式

### 禁止的用法

- 不要链接到不存在的笔记（除非有意创建占位笔记）
- 不要在同一个段落中密集链接（影响阅读）
- 不要用 wikilink 替代必要的解释文字
