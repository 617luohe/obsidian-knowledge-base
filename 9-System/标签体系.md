# 标签体系

## 设计原则

- **层级制**：使用 `/` 分隔层级（如 `type/resource`）
- **互斥性**：一条笔记只有一个 `type/` 标签
- **前缀驱动**：用前缀区分标签的语义维度

## 标签维度

### type/ — 笔记类型（必选，单选）

```
#type/moc           — MOC 索引笔记
#type/project       — 项目笔记
#type/area          — 责任域笔记
#type/resource      — 原子知识笔记
#type/journal       — 日记
#type/person        — 人物笔记
#type/source        — 来源笔记
#type/template      — 模板
#type/meeting       — 会议笔记
#type/spec          — 规范文档
```

### status/ — 生命周期（必选，单选）

```
#status/inbox       — 待处理（仅 0-Inbox 使用）
#status/active      — 进行中
#status/paused      — 暂停
#status/completed   — 已完成
#status/archived    — 已归档
#status/draft       — 草稿（笔记内容待完善）
#status/published   — 已完成发布
```

### confidence/ — 笔记成熟度（仅 #type/resource 使用）

```
#confidence/seed       — 种子：初步想法，可能不完整
#confidence/sprout     — 发芽：内容充实，有一些链接
#confidence/evergreen  — 长青：完善、链接丰富、持续维护
```

### domain/ — 知识领域（可多选，按需扩展）

```
#domain/tech
#domain/tech/ai
#domain/tech/backend
#domain/tech/frontend
#domain/tech/devops
#domain/tech/data

#domain/business
#domain/business/strategy
#domain/business/marketing
#domain/business/management

#domain/science
#domain/humanities
#domain/humanities/psychology
#domain/humanities/philosophy

#domain/health
#domain/finance
#domain/design
#domain/writing
```

### 特殊标签

```
#system/convention   — 知识库规范文件
#system/skill        — 技能定义文件
#system/home         — 入口/主页文件
```

## 标签使用规则

1. **type/ 和 status/ 对所有笔记必填**
2. **confidence/ 仅对 Resources 目录下的笔记使用**
3. **domain/ 可选，但强烈建议填写**，用于按领域筛选
4. **标签全部小写**，使用英文
5. **同一个维度的标签单选**（如不能同时有 `#status/active` 和 `#status/archived`）

## 维护

标签体系应保持精简。添加新 domain 标签前，确认：
- 该领域至少有 5 篇以上的相关笔记
- 该领域与现有 domain 不重叠
